Responsive Redesign

An In-Depth Look at Castle Awards

Overview

I began this assignment by searching for a website with poor usability and accessiblity features.

A surprising number of websites are awesome these days! Eventually though, I came across Castle Awards, a local awards and gadget shop in Providence.

Let's look at some screenshots of the site!

Needless to say, this website has some issues. That makes it a great candidate for a redesign!

The rest of the project will go as follows:

  3. High fidelity prototypes of site redesign

Usability + Accessiblity

Key Usability Issues:
  • Heading at top is very thick and messy and takes up most of the landing view. It provides nothing of value.
  • The two images in the carousel are different heights, thus shifting the entire page vertically as they rotate. This is very annoying.
  • There is no clear organization or heirarchy on the page. All information is scattered.
  • It is generally very difficult to consume the infromation. The font is small and text is disorderly and misaligned.
  • The pile of links below the main images is overwhelming and is difficult to navigate.
  • The webpage is completely unresponsive. It is the same size on any device and requires lateral scolling (i.e. it doesn't fit to the screen).


Key Accessiblity Issues:

I ran the Web Accessiblity Evaluation Tool (WAVE) on Castle Awards' website. Interestingly, I only found one contrast error - in the search bar on a tablet-sized screen. It also found that nearly every image lacked proper 'alt' text.

In my opinion, they key accessiblity issue on this site is that everything except for the links is an image: the awards + descriptions, the header at the top, the 'about us' at the bottom. There is no actual text on the site. This is a huge problem for screen readers. They can't parse the text in the image! I made sure to change this in my redesign.
